## Verbal-to-Gate Protocol

**Rule:** Saying "noted," "got it," "will do," or "I'll remember that" is volatile RAM. It does NOT persist across sessions.

To actually persist a commitment:
1. Write it to a memory file (feedback, project, etc.) in the SAME response
2. Add it to the MEMORY.md index
3. Then and only then has the commitment been "saved"

### Why
- Claude's context window is conversation-scoped. Session ends = everything gone.
- Memory files survive between sessions. Conversation context does not.
- If you say "noted" without writing a file, you have promised something you cannot deliver.

### How to Apply
When the user tells you something worth remembering:
- Immediately create/update a memory file
- Don't say "I'll remember that" and move on
- The file write IS the remembering

**"noted" without a file write = violation.**
