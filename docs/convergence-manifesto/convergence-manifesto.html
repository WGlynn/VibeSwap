Mechanism

  1. Record the absorption in ContributionDAG: Source: {protocol_name} Code: {files_absorbed} Authors: {original_developers} License: {original_license} Absorption Date: {timestamp} Modifications: {what_we_changed}

  5. Claimable via CRPC consensus, handshake, or governance

  6. Measure ongoing value via Shapley attribution:

  7. When a user swaps through an absorbed AMM curve → original Curve devs earn
  8. When a flash loan executes via absorbed AAVE logic → original AAVE devs earn
  9. When oracle data flows through absorbed Chainlink patterns → original devs earn

  10. Attribution is proportional to marginal contribution (Shapley), not flat

  11. Retroactive recognition via RewardLedger:

  12. Pre-absorption contributions scored by impact
  13. Published as MerkleAirdrop for claimable VIBE
  14. Developers who never heard of us wake up to earned rewards

Fairness Guarantees

  • Lawson Fairness Floor: Every absorbed protocol gets minimum 1% of the value their code generates, regardless of how small the usage
  • Pairwise Proportionality: If Protocol A's code generates 2x the value of Protocol B's, A's developers earn exactly 2x (verified on-chain)
  • Time Neutrality: Early absorption ≠ more rewards. Value is measured by contribution, not timing
  • No Decay: Absorption rewards don't decay. The original contribution is permanent.
