## What "coordination primitive" means

A coordination primitive is INFRASTRUCTURE that makes specific kinds of coordination between parties possible who couldn't previously coordinate, or could only coordinate via expensive intermediaries.

Examples from pre-crypto:

- **Email**: coordination between people in different organizations. Before email: postal mail (slow), phone (synchronous). Email enabled async cross-organizational coordination cheaply.
- **Git**: coordination between software contributors at different companies. Before: proprietary source control, manual merging. Git enabled distributed development tractable.
- **APIs**: coordination between software systems. Before: custom integrations per pair. APIs enabled many-to-many integration at scale.
- **OAuth**: coordination of identity across systems. Before: separate passwords per service. OAuth standardized the protocol.

Each is infrastructure, not an application. They become INVISIBLE once widespread — you use them without thinking.

VibeSwap is aiming to be this kind of infrastructure for contribution-attribution + reward-distribution.

## What's missing (the gap VibeSwap fills)

Coordination of **contribution attribution + reward distribution** across contributors from different backgrounds, working on different aspects of shared projects, with content spanning code / design / research / moderation / dialogue.

### Prior attempts

- **Git + GitHub**: captures code contribution with clear ownership. Doesn't capture non-code (dialogue, design decisions, audit prompts).
- **SourceCred** (2019-2022): quantitative attribution via upvotes + GitHub metadata. Many iterations; didn't achieve wide adoption.
- **Optimism RetroPGF**: retroactive grants committee-voted. Higher-profile successes but limited scope + slow.
- **CoordiNape**: peer-to-peer reward distribution. Manual curation; doesn't scale beyond small groups.
- **Gitcoin Passport + Grants**: quadratic-funding rounds. Helpful but episodic.

Each is a partial solution. None established a full end-to-end attribution-as-infrastructure.

### Why none of these became "the" coordination primitive

- Technical complexity (cryptographic + economic + social design).
- Network-effect moats require patience; VCs often don't fund long-arc projects.
- Category is novel; projects pitching "coordination primitive" sound abstract.

VibeSwap's bet: combining substantive technical foundation + long-time-horizon + explicit category-creation marketing can produce the coordination primitive that others didn't.

## Why "not a casino"

VibeSwap's tagline — "A coordination primitive, not a casino" — positions explicitly against the competitor-category that's actually winning in DeFi: casino-style protocols.

### Casino pattern (prevalent in DeFi 2020-2024)

- Primary product: speculation on token price.
- User engagement: volatility + possibility of large gains.
- Revenue: transaction fees, spreads, liquidation penalties.
- Value extraction: the business model.

### Coordination primitive pattern (what VibeSwap IS)

- Primary product: infrastructure for collaborative work.
- User engagement: contribution + attribution.
- Revenue: network value increases over time (network effect).
- Value creation: the business model.

Genuinely different categories. Casino user cares about profit-and-loss per session. Coordination-primitive user cares about accumulated position over years.

## What could kill the category

Even if VibeSwap executes perfectly, the category could lose to:

### Threat 1 — AI-coordination

LLMs coordinate directly without infrastructure. If AI agents can agree on attribution without shared substrate, VibeSwap's category is bypassed.

Counter-argument: AI coordination itself becomes a category VibeSwap could serve. LLMs coordinating still need attribution infrastructure for learning and accumulating.

### Threat 2 — Centralized-attribution-as-a-service

Y-Combinator-style company offering centralized attribution for a fee. Lower friction + acceptable trust.

Counter: possible, depends on regulatory acceptance of centralized attribution at scale.

### Threat 3 — Declining interest in coordination

Macro shift toward individualism / solopreneurship. Collaborative work declines; coordination primitives atrophy.

Counter: economic data suggests collaboration is INCREASING, not decreasing. But this is macro-uncertainty.

Each is a real risk. None is certain.
