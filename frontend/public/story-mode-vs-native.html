Story Mode vs native Claude Code

An honest comparison. The pieces already ship in the box — Story Mode is what you build out of them.

The one-line. Claude Code already has the menu and the fork as features. Story Mode makes them the interface. Not a new capability — a standing pattern that turns an occasional decision-popup into the whole steering wheel: personalized, default-on, and self-tuning.
10 branches · every turn · one keystroke

What Story Mode is. The original game-AI loop — Zork, the dialogue tree, choose-your-own-adventure — running on an LLM. The dungeon master went from a lookup table to a mind, and the dungeon became your actual project. At the end of every turn the agent appends a ranked menu of the most likely next moves, drawn from your decision history. You reply with a number. The branches are generated live; the choices do real work.

What each one actually is

Native Claude Code

Ships the primitives — the menu, the plan gate, the rewindable branch tree. Real, in the product today.

  • AskUserQuestion — a 2–4 option labeled menu, fired when the agent hits a real decision
  • Plan mode — propose a plan, you approve or reject before it acts
  • Rewind / checkpoints — jump back to an earlier message and take a different path
  • Slash commands — user-defined shortcuts you invoke by name

Story Mode (overlay)

A standing interface built on those primitives. Hook-driven, not a fork of the product.

  • The menu ends every turn — default interface, not an occasional interrupt
  • 10 options ranked from your own decision corpus, not 2–4 the model invents on the spot
  • One-keystroke forward navigation (3 or 5,4,1) drives a long run
  • A self-learning loop: it measures hit-rate against your real picks and reweights toward your hand

"Can't I just use the native widget?"

The obvious move is to render the AFK menu through AskUserQuestion and get Anthropic's polished selectable component for free. It doesn't work, because the widget and its constraints are a package deal — a hook can only emit text, so the only way to get the native picker is to call the tool, and the tool brings:
  • A hard cap of 4 options. The menu is 10 ranked moves. You'd have to amputate the wheel down to 4.
  • It blocks instead of garnishes. AFK is ignorable — type anything and move on. The native tool stops and demands a selection.
  • It's model-discretion, not deterministic. AFK fires from a hook every turn, guaranteed. The tool appears only if the model chooses to call it.
  • The learning loop breaks. Native picks don't flow into the corpus-reweighter, so the menu stops getting better at being you.
So the right design is two layers, not a swap: keep the AFK 10-menu as the always-on steering wheel, and call AskUserQuestion only for genuine 2–4-way forks where its UX shines and its limits don't bite. Native polish where it fits; Story Mode everywhere else.

How it knows it's working

catch-rate · recall@10 · the number that matters

A self-tuning menu needs a loss function or it just drifts. Story Mode's primary metric is catch-rate: of every turn where a menu was on offer, how often your actual next move was one of the ten — not "did you pick something," but was the right branch even there. Secondary is precision@1–3: when caught, did it rank near the top. And there's a deliberate guard against the obvious trap — maximizing raw pick-rate would push the menu toward bland, always-applicable filler, so a hit that lands only on a generic standing move counts as a miss, not a win. The loop logs every turn, measures catch-rate, and reweights toward the branches you actually take.

It composes — it isn't a replacement

Story Mode is not a fork of Claude Code and was never trying to out-feature it. Anthropic shipped the menu primitive (AskUserQuestion) and the branch primitive (rewind) — that's the hard part, and it's done. Story Mode answers a narrower question: what if the menu were the default way you steered a long autonomous run, ranked from how you actually decide, and it got better at being you the more you used it?

You don't switch off native Claude Code to get this — you layer it on top. The primitives stay native; the standing interface is the overlay. Augment, don't replace.
