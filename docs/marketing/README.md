# Marketing

> Pitch decks, content pipelines, social, press, explainers.

## What lives here

Public-facing material organized by surface (deck, Medium, dev.to, social, forum) and by stage (queue, pipeline, rollout). Strategy folders carry the playbooks and frameworks behind the output. If you want the "why" of the work being marketed, see [`../research/`](../research/) and [`../concepts/`](../concepts/). If you're a partner evaluating a deck, [`../partnerships/`](../partnerships/) carries partner-specific versions.

## Subfolders

- `pitch/` — pitch decks (2026-deck is current)
- `medium/` — Medium articles, rollout plan, paper-pipeline, content pipeline
- `devto/` — dev.to articles
- `social/` — LinkedIn, Twitter/X, Reddit, TikTok queues and drafts
- `forums/` — ethresearch, Hacker News, Nervos forum posts
- `explainers/` — ELI5 and user-facing explainers
- `announcements/` — JUL API bridge, Shard miner launch
- `strategy/` — playbooks, monetization frameworks, prompt strategies
- `bd-output/`, `content-queue/` — BD output and the content queue

## When NOT to look here

- The actual research being marketed → [`../research/`](../research/) (whitepapers, theorems, essays)
- Per-partner sales artifacts → [`../partnerships/`](../partnerships/)
- Frontend visual design (terminal-console aesthetic) → project root [`CLAUDE.md`](../../CLAUDE.md)
- Internal session reports / RSI / KPIs → [`../_meta/`](../_meta/)
