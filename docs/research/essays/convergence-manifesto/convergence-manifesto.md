## The Shapley Fairness Protocol for Absorbed Code

### Principle

> "Every protocol on this list solved a real problem. Then they put a tollbooth in front of the solution. We remove the tollbooth and replace it with math."

### Mechanism

When code or ideas are absorbed from an external protocol:

1. **Record the absorption** in ContributionDAG:
   ```
   Source: {protocol_name}
   Code: {files_absorbed}
   Authors: {original_developers}
   License: {original_license}
   Absorption Date: {timestamp}
   Modifications: {what_we_changed}
   ```

2. **Create placeholder accounts** for original developers via PlaceholderEscrow:
   - Identity anchored to their GitHub/platform username
   - VIBE accrues based on usage of their absorbed code
   - Claimable via CRPC consensus, handshake, or governance

3. **Measure ongoing value** via Shapley attribution:
   - When a user swaps through an absorbed AMM curve → original Curve devs earn
   - When a flash loan executes via absorbed AAVE logic → original AAVE devs earn
   - When oracle data flows through absorbed Chainlink patterns → original devs earn
   - Attribution is proportional to marginal contribution (Shapley), not flat

4. **Retroactive recognition** via RewardLedger:
   - Pre-absorption contributions scored by impact
   - Published as MerkleAirdrop for claimable VIBE
   - Developers who never heard of us wake up to earned rewards

### Fairness Guarantees

- **Lawson Fairness Floor**: Every absorbed protocol gets minimum 1% of the value their code generates, regardless of how small the usage
- **Pairwise Proportionality**: If Protocol A's code generates 2x the value of Protocol B's, A's developers earn exactly 2x (verified on-chain)
- **Time Neutrality**: Early absorption ≠ more rewards. Value is measured by contribution, not timing
- **No Decay**: Absorption rewards don't decay. The original contribution is permanent.
