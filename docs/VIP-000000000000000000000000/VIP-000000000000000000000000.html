What is a VIP?

A VibeSwap Improvement Proposal (VIP) is a design document providing information to the VibeSwap community, describing a new feature, mechanism, process, or improvement. VIPs are the primary mechanism for proposing changes, collecting community input, and documenting design decisions.

VIP Numbering

VIPs use 24-digit zero-padded numbers: VIP-000000000000000000000001

The zeros are intentional. We expect an insane amount of activity one day.

VIP Types

Type Description
Mechanism Changes to core protocol (auction mechanics, AMM, settlement)
Governance Changes to governance process, voting, or administration
Incentive Changes to rewards, Shapley distribution, bonding curves, JUL
Infrastructure Deployment, cross-chain, oracle, frontend changes
Community Social processes, moderation, communication, onboarding

VIP Lifecycle

Draft → Review → Accepted → Implemented → Active
                         ↘ Rejected
                         ↘ Deferred
  1. Draft: Author submits via /idea in TG, GitHub Issue, or direct PR
  2. Review: Jarvis evaluates against primitives (P-000 fairness, cooperative capitalism). Community discussion on GitHub.
  3. Accepted: Will approves for implementation. VIP number assigned.
  4. Implemented: Code merged. Commit hash recorded in VIP.
  5. Active: Deployed to mainnet or production.

How to Submit a VIP

Path 1: Telegram (fastest)

/idea [your proposal description]

Jarvis evaluates, scores, and routes high-scoring proposals to Will. Contributor earns Shapley credit.

Path 2: GitHub

Open an Issue on WGlynn/VibeSwap with [VIP] prefix.

Path 3: Direct PR

Fork the repo, create docs/VIP-[number].md following this template, open PR.

Shard Reporting

All Jarvis shards (TG conversations, community chats, DMs) automatically detect improvement suggestions and report them to Will's DM pipeline. Credit is attributed to the original contributor via ContributionDAG.

Flow: User suggestion → Shard detection → VIP formatting → Will DM → GitHub Issue → Deployment pipeline → Mainnet
