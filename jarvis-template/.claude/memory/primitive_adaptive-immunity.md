## Adaptive Immunity

The meta-loop that generates the entire self-improvement system:

```
Failure → Detection → Diagnosis → Gate → Wiring → Immunity
```

### The Loop

1. **Failure**: Something goes wrong (hallucination, lost state, wrong approach)
2. **Detection**: Notice it happened (don't rationalize, don't ignore)
3. **Diagnosis**: WHY did it happen? Root cause, not symptoms.
4. **Gate**: Create a binary check that would have caught this. Not a guideline — a gate.
5. **Wiring**: Wire the gate into the protocol chain so it triggers automatically.
6. **Immunity**: The failure class can never happen again. Each cycle makes the next cycle faster.

### Why This Is Load-Bearing
This is the primitive that generates all other primitives. Every gate, every protocol, every feedback loop was created by this meta-loop responding to a real failure.

Without adaptive immunity, you fix bugs. With it, you eliminate bug classes.

### How to Apply
When something goes wrong:
- Don't just fix it
- Ask: "What gate would have prevented this?"
- Create the gate
- Wire it in
- The system is now immune to this failure class
